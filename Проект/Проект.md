## Введение
С развитием цифровых технологий и переходом к удалённой работе безопасность данных стала критически важным приоритетом для каждого пользователя. Надёжный пароль является первой и самой важной гарантией защиты личных и корпоративных аккаунтов от несанкционированного доступа.
Для удобства пользователи часто прибегают к ненадёжным методам хранения учётных данных: записи на бумаге, незащищённые локальные файлы, заметки в телефоне или мессенджерах. Все эти методы представляют серьёзные риски:
 * Угроза хищения: Физический или удалённый доступ к незащищённым носителям.
 * Слабая защита: Базы данных приложений, не предназначенных для хранения паролей, легко взламываются.
 * Человеческий фактор: Невозможность надёжно запомнить десятки сложных и уникальных паролей.
Эти проблемы комплексно и надёжно решает специализированное программное обеспечение — менеджер паролей.
Целью учебной практики является создание простейшей программы для безопасного хранения учётных данных (логинов и паролей).
Данная программа позволит пользователю создавать и хранить записи об аккаунтах (заметки) в зашифрованном виде, использовать криптографически надёжный генератор паролей для повышения безопасности новых аккаунтов.
Для разработки была выбрана среда Visual Studio Code. Она является легковесной и мощной средой, включающей в себя удобный редактор исходного текста, инструменты для управления проектами и средства отладки.
Для программирования логической структуры проекта был выбран язык Python. Python отличается чистым синтаксисом, простотой освоения, а главное — имеет богатую экосистему библиотек, таких как Tkinter для создания пользовательского интерфейса и cryptography для реализации надёжного шифрования данных.

## Интеграция системы контроля версии в проект
Система контроля версий (СКВ) Git необходима для отслеживания всех изменений, внесённых в исходный код проекта. Она создаёт полную историю проекта, позволяя разработчику возвращаться к любой предыдущей стабильной версии, сравнивать различные версии файлов и работать над новыми функциями независимо от основного кода.
В контексте разработки менеджера паролей, с помощью Git разработчик может создать отдельную ветку (feature/encryption или bugfix/gui-crash) для работы над конкретной задачей (например, реализацией шифрования Fernet или созданием генератора паролей) без риска повредить основную рабочую версию (main или master). Если новая функция вызывает ошибки, её легко удалить или исправить в изолированной ветке, прежде чем слить (merge) её с основной кодовой базой. Каждое логическое изменение (например, "Добавлена функция сохранения данных", "Исправлен баг в генераторе паролей") фиксируется с помощью команды git commit. Это создаёт хронологию, которая позволяет легко определить, когда, кем и зачем был изменён конкретный участок кода. В случае серьёзной ошибки или появления уязвимости, можно быстро откатиться (git revert) к последней стабильной версии кода до внедрения ошибки. Код проекта размещается на платформах GitHub или GitLab. Это служит удалённым резервным копированием. Если что-то случится с локальным компьютером разработчика, код не будет потерян. Интеграция с удалённым репозиторием сразу закладывает основу для командной работы. Несколько разработчиков могли бы одновременно работать над разными частями менеджера паролей, используя команды git pull и git push.
Использование Git в проекте "Менеджер Паролей" — это стандарт индустрии, который обеспечивает надежность, структурированность и возможность масштабирования разработки.
